Tipos de Dados

Em boa parte das linguagens de programação, é necessário informar o tipo de dado que será armazenado dentro da variável. Assim, cada tipo armazena um formato e carrega suas peculiaridades, sendo os principais: 

                    
                | TIPO   | CONCEITO             | *REPRESENTAÇÃO | DECLARAÇÃO                 |
                | ---    |  --- | --- | ---  | -|                |                            |
                | int    | Números inteiros     | %i             | int x                      |
                | float  | Números decimais     | %f             | float x                    |
                | char   | 1 Caractere          | %c             | char x;                    |
                | string | Cadeia de caracteres | %s             | char nomeVariavel[tamanho] |

Eletrônica Para Arduino

O Arduino é uma plataforma de prototipagem eletrônica amplamente utilizada, especialmente na área de robótica e projetos eletrônicos. Aqui estão alguns conceitos de eletrônica relacionados ao Arduino:

  1. Microcontrolador: O Arduino é baseado em um microcontrolador, que é um chip integrado que contém um processador, memória e periféricos de entrada/saída. O microcontrolador é o "cérebro" do Arduino e executa o código que controla o funcionamento do projeto.
  2. Pinos de E/S: O Arduino possui pinos de entrada/saída (I/O) que permitem a comunicação com outros dispositivos eletrônicos. Esses pinos podem ser configurados como entrada (para receber informações) ou saída (para enviar informações).
  3. Tensão de Alimentação: O Arduino geralmente é alimentado por uma fonte de energia externa, como uma bateria ou uma fonte de alimentação. É importante entender a tensão de alimentação suportada pelo Arduino e fornecer a tensão adequada para seu correto funcionamento.
  4. Pinos Digitais: Os pinos digitais do Arduino são utilizados para sinais binários, onde podem ser configurados como entrada ou saída. Eles são usados para enviar e receber sinais de nível lógico alto (5V) ou baixo (0V).
  5. Pinos Analógicos: O Arduino também possui pinos analógicos que são usados para entrada de sinais analógicos. Eles permitem a leitura de valores contínuos, como sensores de luz, temperatura, umidade, entre outros.
  6. PWM (Modulação por Largura de Pulso): O Arduino tem a capacidade de gerar sinais PWM nos pinos digitais. A modulação por largura de pulso é utilizada para controlar a intensidade de um sinal, permitindo o controle de brilho de um LED, velocidade de um motor, entre outros.
  7. Comunicação Serial: O Arduino suporta comunicação serial, permitindo a troca de dados com outros dispositivos. Isso pode ser feito por meio dos pinos de comunicação serial (RX, TX) ou por meio de interfaces como I2C e SPI.
  8. Bibliotecas: O Arduino possui uma ampla variedade de bibliotecas que fornecem funções e recursos adicionais para simplificar o desenvolvimento de projetos. Essas bibliotecas incluem funcionalidades para controle de motores, leitura de sensores, exibição em displays, entre outros.
  9. Shields: Os shields são placas de expansão que se encaixam no topo do Arduino e fornecem funcionalidades adicionais, como conectividade Wi-Fi, Ethernet, GPS, controle de motores, entre outros. Eles facilitam a adição de recursos extras ao Arduino.
  10. Programação: O Arduino é programado usando a linguagem de programação C/C++. É importante ter familiaridade com conceitos básicos de programação, como variáveis, estruturas de controle (if, loops), funções, entre outros.

Usando C no arduino

A linguagem Arduino é uma derivação da linguagem C++, portanto, a maioria das sintaxes básicas são semelhantes. No entanto, existem algumas diferenças importantes a serem observadas:

  1. Função setup() e loop(): No Arduino, o código principal é dividido em duas funções principais: setup() e loop(). A função setup() é executada apenas uma vez, no início do programa, e é usada para realizar as configurações iniciais, como inicializar pinos, definir velocidades de comunicação, etc. Já a função loop() é executada repetidamente em um loop infinito após a função setup() e é onde você escreve o código principal do seu programa, que será executado continuamente.
  2. Função analogRead(): No Arduino, para ler os valores de uma porta analógica, você usa a função analogRead() em vez da função scanf() ou fscanf() usadas em C. A função analogRead() retorna um valor inteiro representando a leitura analógica.
  3. Função pinMode(): Para configurar um pino como entrada ou saída no Arduino, você utiliza a função pinMode(). Essa função permite definir a direção do fluxo de dados em um pino específico.
  4. Funções específicas do Arduino: O Arduino possui uma série de funções e bibliotecas específicas para acessar seus recursos, como leitura de sensores, controle de atuadores, comunicação serial, PWM, entre outros. Essas funções são exclusivas do ambiente Arduino e podem não estar disponíveis na linguagem C padrão.

Embora existam essas diferenças na sintaxe e no ambiente de programação, o Arduino ainda é baseado na linguagem C++ e muitos dos princípios e conceitos fundamentais da linguagem C também se aplicam no contexto do Arduino. Portanto, ter um conhecimento básico de C será útil ao programar para o Arduino.

Estrutura de Decisão

A estrutura condicional permite que o programa execute diferentes ações com base em condições específicas. Assim:

- Se a condição for atendida (T), o programa segue um caminho;

- Se a condição não for atendida (F), o programa segue outro caminho.

Diante dessas circunstâncias, é possível destacar as principais estruturas de decisão:

IF

Irá realizar algo caso a sua condição retorne true. 

            
                1       int a = 10;
                2       
                3       void setup() {
                4         Serial.begin(9600);
                5       }
                6       
                7       void loop() {
                8         if (a == 10) {
                9           Serial.println("A condição é verdadeira!");
                10         }
                11       delay(1000);
                12     }

ELSE IF

Múltiplas condições em forma de cascata, o else if adiciona mais um condição caso o if anterior retorne false, permitindo utilizar um else no final para exceções. 

                        
             1       int a = 10;
             2       
             3       void setup() {
             4         Serial.begin(9600);
             5       }
             6      void loop() {
             7       if (a < 10) {
             8        Serial.println("a é menor do que 10");
             9       } else if (a > 10) {
             10       Serial.println("a é maior do que 10");
             11      } else if (a == 10) {
             12       Serial.println("a é igual a 10");
             13      } else {
             14       Serial.println("a é um valor inválido!");
             15      }
             16       delay(1000);
             17      }

Ao rodar o exemplo, será obtido o terceiro print, “A é igual a 10”, o código irá rodar as condições uma a uma, porém como a variável a não é maior nem menor do que 10, elas retornaram false, enquanto a condição de igual retornará true e executará o código em seu interior.

Switch-Case

Estrutura de escolha ou seleção utilizada para selecionar uma opção verdadeira, isto é, quando é preciso uma quantidade maior de condições. Além disso, uma de suas exclusividades é aceitar critérios apenas do tipo inteiro ou char.

A instrução break associada a cada case garante que o programa saia da condicional switch e execute a instrução que segue logo após a switch. Caso o break seja omitido, o programa continua a execução para a próxima instrução dentro do switch (até encontrar um “break”) - se nenhum “break” for encontrado, o programa pode executar códigos indesejados ou até entrar em um loop infinito. 

            
             1   int input;
             2
             3  void setup() {
             4    Serial.begin(9600);
             5  }
             6   
             7  void loop() {
             8    switch(input){
             9      case 1: 
             10       Serial.println("Opção 1.");
             11     case 2: 
             12       Serial.println("Opção 2.");
             13     case 3: 
             14       Serial.println("Opção 3.");
             15     default:
             16       Serial.println("Opção reserva.");
             17   }
             18   delay(1000);
             19 }
